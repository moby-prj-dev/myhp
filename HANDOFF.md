# 沖縄デジタルワークス コーポレートサイト 引き継ぎメモ

## このサイトの目的
- 沖縄デジタルワークス（個人事業 / 屋号）の **事業者としての信用づくり** が最優先目的。
- 下請け営業時に「沖縄デジタルワークスとは何者か」を検索された際の **名刺代わりのページ**。
- 派手さより「実在するまともな事業者だ」と一瞬で伝わることを重視。盛らないことが信用につながる方針。
- 構成は **1ページ完結（縦スクロール）**。あえてページを増やさず、1スクロールで全部わかる作りにしている。

## ポジショニング（※重要・盛らない）
- ~10年の業務システム開発経験（C / VB / Baan ERP 等）＋近年 Laravel / PHP に注力。
- 役割は **実装・コーディング**。提供された仕様・設計をもとに作るのが軸。
- **NG表現**: 「Laravel歴10年以上」「要件定義から対応」など、事実を超える表現は使わない。
- 外部向け資料では **沖縄デジタルワークスのみ** に言及（他の所属には触れない）。

## 技術構成（現状）
- 単一の `index.html`。フレームワークなし、ビルド不要のプレーンなHTML + CSS + 最小限のJS。
- フォント: Google Fonts（`Zen Kaku Gothic New` / `Outfit` / `JetBrains Mono`）。
- JSは2つだけ:
  1. スクロールでヘッダーに枠線を出す
  2. `.reveal` 要素を IntersectionObserver でフェードイン
  3. FAQアコーディオンの開閉
- 外部ライブラリ依存なし。そのままブラウザで開けば動く。

## デザイン方針
- テーマ: 落ち着いた **濃色（ダーク）ベース** の refined / minimal。
- カラーはCSS変数で管理（`:root`）。主要なもの:
  - `--bg: #0f1419`（ベース背景）
  - `--accent: #4db8a8`（ティール系アクセント）
  - `--ink: #e8edf2`（本文）
- セクションごとにテーマに合った背景レイヤー（`.sec-bg`）を持たせ、表情を変えている。
  すべて radial/linear-gradient にマスクをかけ、中央だけにさりげなく効かせる（ベタ塗りで安っぽくしない）。

   - 求人マッチングプラットフォーム（Laravel / MySQL / Docker / 外部API）
   - 店舗向け業務管理システム（Laravel / Blade / Alpine.js / MySQL）

5. **03 開発の進め方（Flow）** — ヒアリング→仕様確認→実装→テスト→納品 の5ステップ。背景=横方向グラデ
6. **04 経歴・スキル（Profile）** — 経歴文＋スキル一覧＋資格（基本情報技術者）
7. **05 技術スタック（Tech）** — Backend / Frontend / Infra の3カテゴリ。背景=ターミナル風スキャンライン
8. **06 事業者情報（Information）** — 屋号 / 代表者 / 所在地 / 事業内容 / 開業 の表
9. **07 よくあるご質問（FAQ）** — クリック開閉。実装が軸というポジショニングを正しく伝える内容
10. **問い合わせ（Contact）** — メールアドレスへのリンク
